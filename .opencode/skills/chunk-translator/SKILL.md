# Chunk Translator

将英文纯文本字幕块翻译为中文，采用「整体翻译→智能拆分」两步法，确保翻译流畅自然且行数严格对齐。

## 使用场景

用户有一个从 SRT 字幕提取出的英文纯文本文件（每行一句/一段，无时间码和序号），需要将其翻译为中文，输出为一个行数完全相等的纯中文文本文件。该中文文件将作为中间产物，后续被合并回字幕格式用于中英对照学习。

## 核心原则

### 为什么先整体翻译再拆分？

字幕文本由口语碎片组成，逐行翻译会丢失上下文导致译文生硬、不连贯。将全文合并为一段完整篇章后整体翻译，AI 能理解全文语境，产出自然流畅的中文。然后再按原文行数智能拆分，保证行行对应。

### 为什么用意译而非直译？

英文口语中常有口误、不完整句、重复表达（如 "And by the way, in g by game I mean..."），直译会让中文也充满语病。意译在保持原意的前提下，用自然中文口语重新表达，更适合作为学习对照材料。

## 执行流程

### 第1步：读取与合并

1. 读取英文输入文件，记录总行数 N
2. 将所有行合并为一段连贯的英文篇章
3. 字幕文本常有句子跨行拆分，合并时需自然衔接，补全不完整的句子片段
4. 记录每行英文的完整内容（需要用于后续拆分对齐）

### 第2步：整体翻译

将合并后的英文篇章翻译为流畅中文，遵循以下翻译原则：

- **意译优先**：用自然中文口语表达，不拘泥于字面翻译
- **修正口误**：英文原文的口误、重复、不完整表达，在中文里自然修正——不要说"我的意思是说，我是说RPG"，而是直接说"我指的是RPG"
- **保持语气**：保留原文的疑问、感叹、迟疑、反问等语气特征
- **术语一致**：RPG、类型等专有术语保持翻译一致性，不要一会翻译为"角色扮演游戏"一会简称"RPG"，统一使用
- **风格**：口播视频风格，亲切自然的中文口语，像是中国UP主在说话

### 第3步：智能拆分

将完整中文译文拆分为恰好 N 行，每行中文对应一行英文原文。这是最关键也最容易出错的步骤。

**拆分方法**：

1. 逐行回顾英文原文：理解每行英文的核心含义
2. 在完整中文译文中逐行定位：找到每行英文含义对应的中文段落位置
3. 将中文译文按英文行边界切分为 N 个语义段落，确保：
   - **行数严格等于 N**——不可多、不可少
   - **语义对应**——第 i 行中文表达的是第 i 行英文的含义
   - **拆分点自然**——在中文的自然断句处（句号、问号、逗号、语气词后）切分

**重要提醒**：

- 拆分后的每行中文不必是完整句子——英文一行可能只是半个句子，对应的中文也只需是半个句子
- 多个英文行可能合起来才构成一个完整中文句——没关系，关键是对应
- 空行（如果英文有）在对应中文位置也应保留为空行
- 拆分不是机械切分——必须根据含义对齐，每行中文与同位置的英文行语义一致

### 第4步：校验与输出

1. 数一遍中文行数，确认严格等于 N
2. 如果不等，回到第3步检查哪些行被漏拆或多拆，修正后重数
3. 将 N 行中文写入输出文件：`原文件名_chinese.txt`
4. 输出文件为纯文本，不含序号、空行分隔符、格式标记——每一行就是一句中文翻译

## 输出格式示例

输入文件 (`sample.txt`，5行英文)：
```
Hi everyone, it's me Tim.
Today I want to ask you a question.
Would you play a game that?
What I mean by this is I'm going to describe several
game ideas and I'm curious if you would play that game.
```

输出文件 (`sample_chinese.txt`，5行中文)：
```
大家好，我是Tim。
今天我想问你们一个问题。
你会玩这样的游戏吗？
我想说的是，我会描述几个
游戏创意，很好奇你们会不会玩那个游戏。
```

## 关键约束

- **先整体后拆分**：不要在合并前逐行翻译，不要跳过合并步骤
- **行数必须严格相等**：输出行数 N_chinese == 输入行数 N_english，这是最高优先级的质量要求
- **语义对应优先于断句自然**：拆分时如果断句不自然，可以接受；但行数不对或语义错位不可接受
- **纯文本输出**：不在输出文件添加序号、"第X行："等标记
- **禁止修改原文件**：只创建新的中文文件
