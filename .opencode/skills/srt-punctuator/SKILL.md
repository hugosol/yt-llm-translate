# SRT Punctuator

This skill adds proper English punctuation to SRT subtitle files generated by AI transcription tools (like OpenAI Whisper), which typically produce raw text without any punctuation marks.

## Output naming (do NOT improvise)

After successful completion, the **original input file is replaced** with the punctuated version. During processing, an intermediate file `{original_name}_punctuated.srt` is created as a safety measure — if the finalization completes without errors, the original is deleted and this intermediate file is renamed to the original name.

For example:
- `my_video.srt` → processing → `my_video.srt` (now punctuated, original replaced)
- `/path/to/episode.srt` → processing → `/path/to/episode.srt` (now punctuated, original replaced)

If the script fails or the output is empty, the original file is preserved.

Intermediate working files live in `{original_name}.punc_work/` and are automatically deleted by the script when done — you do not need to clean them up manually.

## Workflow

Exactly three steps. Two script invocations, one punctuation step in between.

### Step 1: Prepare

```bash
python <skill-path>/scripts/srt_marker.py prepare input.srt
```

This creates:
- `input.punc_work/marked.txt` — the full merged text for reference
- `input.punc_work/chunks/` — a directory of chunk files, each with up to 50 entries wrapped in `<<N>>` markers
- `input.punc_work/chunks.json` — manifest: `{"total_chunks": N, "chunk_size": 50, "total_entries": M}`

The script prints something like:
```
Entries: 150  Chunks: 3  Chunk size: 50
Chunks saved to: input.punc_work/chunks/
```

### Step 2: Add punctuation (chunk by chunk)

First, read `input.punc_work/chunks.json` to know the total number of chunks. Then process each chunk one at a time in order.

For each chunk `chunk_NNN.txt` (e.g. `chunk_000.txt`, `chunk_001.txt`, ...):
1. Read `input.punc_work/chunks/chunk_NNN.txt`
2. Add proper English punctuation to the text
3. Write the result to `input.punc_work/chunks/chunk_NNN_punctuated.txt`

Punctuation rules:
- Add appropriate periods, commas, question marks, exclamation marks, colons, semicolons, apostrophes, quotation marks.
- Use context and meaning to decide sentence boundaries.
- Capitalize the first letter after sentence-ending punctuation (`. ? !`).
- Add a space after `. , ? ! : ;` as standard English requires.
- **Critical: NEVER modify the `<<N>>` markers.** They must survive untouched — same digits, same angle brackets. You may add punctuation immediately before or after a marker, but the marker text itself cannot change.
- `[Music]`, `[Applause]` and similar sound-effect tags should be left unchanged.

Each chunk is small enough to process in one go. Work through them sequentially — do not skip any chunk.

### Step 3: Finalize

```bash
python <skill-path>/scripts/srt_marker.py finalize input.srt input.punc_work --from-chunks
```

This reads all `chunk_NNN_punctuated.txt` files in order, concatenates them back into a single punctuated article, splits each entry's text at the `<<N>>` markers, writes `input_punctuated.srt` with the original timestamps and updated punctuated text. If the output is valid (non-empty), it deletes the original `input.srt` and renames `input_punctuated.srt` back to `input.srt`, effectively replacing the original. The `input.punc_work/` directory is then automatically deleted.

The script will refuse to replace the original if the punctuated output is empty or if any chunk is missing its `_punctuated` counterpart (safety check).

### Step 4: Verify

Quickly scan `input.srt` (which is now the punctuated version):
- No stray `<<N>>` markers remain in the text.
- Sentence boundaries have appropriate punctuation.
- Sound-effect tags (`[Music]`, `[Applause]`) are preserved unchanged.
